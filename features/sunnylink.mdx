## Overview

sunnylink is sunnypilot's cloud connectivity platform that provides secure device registration, encrypted settings backup and restore, and future remote management capabilities. Think of it as your sunnypilot account in the cloud, ensuring your configuration is never lost and enabling advanced features.

<Info>
sunnylink uses end-to-end encryption to protect your settings data. All encryption happens on-device before transmission, and only your device can decrypt the backups.
</Info>

## How It Works

### Registration Process

<Steps>
  <Step title="Enable sunnylink">
    Navigate to **Settings → Device → sunnylink** and toggle it on. Your device will automatically register on next boot.
  </Step>
  
  <Step title="Pair GitHub Account">
    Scan the QR code displayed in settings to pair your GitHub account. This enables sponsor tier features and API access.
  </Step>
  
  <Step title="Automatic Setup">
    sunnylink automatically generates a unique device ID and cryptographic keys for secure communication.
  </Step>
</Steps>

### Security Architecture

<Accordion title="Encryption Details">
  **Key Generation:**
  - RSA key pair generated on-device
  - AES-256 encryption key derived from RSA private key
  - Elliptic curve keys supported for enhanced security
  
  **Data Protection:**
  - Settings encrypted on-device before upload
  - Transmitted over HTTPS
  - Stored encrypted on sunnylink servers
  - Only your device can decrypt backups
  
  **Implementation:**
  ```python
  # Settings are compressed and encrypted before upload
  config_json = json.dumps(config_data)
  encrypted_config = encrypt_compressed_data(config_json, use_aes_256=True)
  ```
</Accordion>

### Restoring Settings

<Warning>
Restoring settings will overwrite your current configuration. Make sure you want to proceed before confirming.
</Warning>

<Steps>
  <Step title="Access Restore">
    Navigate to **Settings → Device → sunnylink → Restore Backup**
  </Step>
  
  <Step title="Select Version">
    Choose which backup version to restore (latest is selected by default)
  </Step>
  
  <Step title="Confirm Restore">
    Review the backup details and confirm the restore operation
  </Step>
  
  <Step title="Reboot">
    Device will apply settings and may require a reboot for some changes to take effect
  </Step>
</Steps>

## API Integration

sunnylink provides API access for advanced users and future integrations:

### Authentication

```python
from openpilot.sunnypilot.sunnylink.api import SunnylinkApi

# Initialize API client
api = SunnylinkApi(device_id)

# Get authentication token
token = api.get_token()

# Make authenticated requests
result = api.api_get(
    endpoint="backup/device_id",
    access_token=token
)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Registration fails">
    **Symptoms:**
    - "SunnylinkTempFault" error
    - Device shows unregistered after enabling
    - QR code won't display
    
    **Solutions:**
    
    1. Check internet connectivity (WiFi or mobile data)
    2. Verify device has network access during boot
    3. Check for firewall/proxy blocking sunnylink API
    4. Try disabling and re-enabling sunnylink
    5. Reboot device and wait for automatic registration
    
    **Logs to check:**
    ```bash
    # Check registration logs
    cat /data/community/crashes/registration_manager.log
    ```
  </Accordion>
  
  <Accordion title="Backup fails to create">
    **Possible causes:**
    
    - No internet connection
    - sunnylink not registered
    - API server timeout
    - Encryption error
    
    **Debugging steps:**
    
    1. Verify sunnylink shows as registered
    2. Check internet connectivity
    3. Review backup progress indicator
    4. Try again after waiting a few minutes
    5. Check device logs for specific errors
  </Accordion>
  
  <Accordion title="Restore doesn't apply settings">
    **Check these:**
    
    - Settings were successfully downloaded
    - Restore completed to 100%
    - No error messages during restore
    - Device rebooted after restore
    
    **Note:** Some settings only take effect after reboot. If restore completed successfully but settings seem unchanged, try rebooting your device.
    
    **Version compatibility:** Settings from much newer or older versions may not fully restore if parameter definitions have changed.
  </Accordion>
  
  <Accordion title="GitHub pairing not working">
    **Requirements:**
    
    - sunnylink must be registered first
    - QR code must be scanned with camera app
    - GitHub account must be accessible
    
    **Steps:**
    
    1. Ensure sunnylink is enabled and registered
    2. Navigate to sunnylink panel
    3. Use phone camera to scan QR code
    4. Complete GitHub OAuth flow
    5. Return to device to confirm pairing
  </Accordion>
</AccordionGroup>

## Technical Details

### Backup Manager

The backup system is implemented in `sunnypilot/sunnylink/backups/manager.py`:

```python
class BackupManagerSP:
    """Manages device configuration backups to/from sunnylink"""
    
    # Status tracking
    - backup_status: idle | inProgress | completed | failed
    - restore_status: idle | inProgress | completed | failed
    - progress: 0.0 to 100.0
    
    # Operations
    - create_backup(): Collect, encrypt, upload settings
    - restore_backup(version): Download, decrypt, apply settings
```

### Parameter Filtering

Only parameters marked with `ParamKeyFlag.BACKUP` are included:

- Feature configuration toggles
- Tuning values and offsets
- UI preferences
- Custom settings

**Excluded from backups:**
- Secrets and tokens
- Device-specific identifiers
- Temporary runtime state
- Cache and debug data

### Network Requirements

- **Registration:** One-time HTTPS connection to sunnylink API
- **Backups:** HTTPS upload on demand
- **Restore:** HTTPS download on demand
- **No persistent connection required**
