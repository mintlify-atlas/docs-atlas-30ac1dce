## Overview

MADS (Modular Assistive Driving System) is sunnypilot's complete driving assistance framework that revolutionizes how you interact with your ADAS system. Unlike traditional systems that tie steering and acceleration controls together, MADS allows independent control of lateral (steering) and longitudinal (speed/acceleration) functions.

<Info>
MADS is one of sunnypilot's most popular features, providing a more natural and flexible driving experience while maintaining safety.
</Info>

## How MADS Works

### Independent Control States

MADS introduces a sophisticated state machine with multiple operational states:

- **Enabled**: Full lateral control is active
- **Paused**: System temporarily suspended but ready to resume
- **Soft Disabling**: Gradual transition to disabled state
- **Overriding**: Driver intervention detected but system remains engaged
- **Disabled**: System fully disengaged

### State Transitions

The system intelligently manages transitions between states based on:

- Driver inputs (steering wheel, pedals, buttons)
- Vehicle conditions (gear position, doors, seatbelt)
- Road conditions and safety events
- Cruise control availability

### Steering Behavior on Brake

MADS offers three configurable behaviors when you press the brake pedal:

<Tabs>
  <Tab title="Remain Active">
    Lateral control continues uninterrupted when braking. Best for highway driving where you want steering assist to remain active during speed adjustments.
  </Tab>
  <Tab title="Pause">
    Lateral control temporarily pauses when braking but automatically resumes when conditions are met. Ideal for mixed driving conditions.
  </Tab>
  <Tab title="Disengage">
    Lateral control fully disengages when braking, requiring manual re-engagement. Required for certain platforms like Rivian and some Tesla models.
  </Tab>
</Tabs>

### Silent State Management

MADS intelligently handles common scenarios without disengaging:

- **Door opening** at standstill
- **Seatbelt unlatching** while parked
- **Wrong gear** events (e.g., reverse gear at low speed)
- **Brake hold** activation
- **Park brake** engagement

Instead of disabling completely, MADS transitions to a "paused" state and can resume automatically when conditions normalize.

## Configuration

### Enabling MADS

1. Navigate to **Settings → SP - Controls**
2. Toggle **Enable MADS**
3. Select your preferred steering behavior mode
4. Configure optional features:
   - Main Cruise Allowed (if applicable)
   - Unified Engagement Mode

### Toggle Methods

Depending on your vehicle platform:

<Steps>
  <Step title="Using Factory Buttons">
    - **Hyundai/Kia/Genesis**: Press the LDA button
    - **Toyota/Lexus**: Press and hold the LKAS button (beta)
    - **Other platforms**: Use cruise control buttons
  </Step>
  
  <Step title="Alternative Methods">
    Some platforms support toggling MADS through:
    - Cruise main button (when Main Cruise Allowed is enabled)
    - Set/Resume cruise buttons (Unified Engagement Mode)
  </Step>
</Steps>

## Technical Implementation

### Event Handling

MADS replaces standard openpilot events with specialized versions:

```python
# Standard events become "silent" alerts in paused state
EventName.doorOpen → EventNameSP.silentDoorOpen
EventName.wrongGear → EventNameSP.silentWrongGear
EventName.reverseGear → EventNameSP.silentReverseGear
```

### Safety Integration

MADS maintains full compatibility with comma.ai's safety policies:

- Driver monitoring requirements remain active
- All safety checks continue to operate
- Actuator limits are enforced
- Alternative experience flags properly set in panda

### State Machine Logic

The MADS state machine (`sunnypilot/mads/state.py`) manages transitions with:

- **Priority handling**: USER_DISABLE and IMMEDIATE_DISABLE events take precedence
- **Soft disable timer**: Gradual 3-second transition to disabled state
- **Override detection**: Recognizes driver steering input while maintaining engagement
- **Enable conditions**: Validates NO_ENTRY events before allowing engagement

## Troubleshooting

<AccordionGroup>
  <Accordion title="MADS won't enable">
    **Check these conditions:**
    
    - Cruise control is available (unless Unified Engagement Mode is on)
    - Vehicle is in Drive gear and moving
    - No active safety alerts
    - Main cruise switch is on (for applicable platforms)
    
    **Solution:** Ensure all engagement conditions are met, then press the LKAS/LDA button or set cruise control.
  </Accordion>
  
  <Accordion title="System disengages unexpectedly">
    **Common causes:**
    
    - Brake pressed with Disengage mode enabled
    - Safety event triggered (lane departure, low speed, etc.)
    - Driver monitoring alert
    
    **Solution:** Check your steering mode setting and review any active alerts. Consider switching to Pause or Remain Active mode if braking is the cause.
  </Accordion>
  
  <Accordion title="Can't toggle with factory button">
    **Platform-specific:**
    
    - Some vehicles require specific conditions to toggle (e.g., cruise available)
    - Certain model years may not support factory button integration
    
    **Solution:** Verify your vehicle is in the supported list. Use cruise control engagement as an alternative toggle method.
  </Accordion>
</AccordionGroup>

## Advanced Features

### Main Cruise Toggleable

On supported Hyundai/Kia/Genesis platforms, you can control whether MADS requires the main cruise switch:

- **Enabled**: MADS activation requires cruise main to be on
- **Disabled**: MADS can be toggled independently of cruise main state

### Unified Engagement Mode

This mode allows MADS to engage with cruise control buttons even when:

- Main cruise is off
- Traditional engagement conditions aren't met
- You want single-button MADS activation

<Warning>
Unified Engagement Mode changes how your system responds to cruise control inputs. Ensure you understand the behavior before enabling.
</Warning>
