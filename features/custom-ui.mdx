## Overview

Sunnypilot features a completely redesigned user interface, rewritten from the ground up using Raylib Python, replacing the original Qt C++ implementation. This modernization brings extensive customization options, improved performance, and a more intuitive user experience both on-road and off-road.

<Info>
The UI overhaul represents one of the largest changes in sunnypilot 2025+, touching every aspect of the visual experience from settings panels to onroad displays.
</Info>

## Complete UI Redesign

### Key Improvements

- **Performance:** Smoother rendering and faster response times
- **Maintainability:** Python codebase easier to develop and debug
- **Flexibility:** More customization options and visual effects
- **Consistency:** Unified design language across all screens

## Display Customization

### Brightness Controls

<Tabs>
  <Tab title="Global Override">
    **Force specific brightness level:**
    
    - Override auto-brightness
    - Set custom day/night levels
    - Useful for very bright/dark conditions
    - Independent of ambient sensor
    
    **Configuration:**
    Settings → Display → Global Brightness Override
  </Tab>
  
  <Tab title="Onroad Brightness">
    **Driving-specific brightness:**
    
    - Set brightness while driving
    - Different from offroad brightness
    - Auto-adjust based on time of day
    - Override ambient sensor when needed
    
    **Configuration:**
    Settings → Display → Customizable Onroad Brightness
  </Tab>
</Tabs>

### Interactive Timeout

Control how long the screen responds to touches:

- **Standard:** 30 seconds (default)
- **Extended:** 60 seconds
- **Always Active:** Never timeout
- **Quick:** 10 seconds

<Warning>
Extended "Always Active" may impact battery life on cordless installations.
</Warning>

### Screen Off While Driving

<AccordionGroup>
  <Accordion title="Enable Screen Off">
    **Power saving and distraction reduction:**
    
    Turn off the display while driving after a configurable timeout:
    - Saves power on cordless setups
    - Reduces visual distraction
    - Screen wakes for alerts
    
    **Timer options:**
    - 10 seconds
    - 30 seconds
    - 1 minute
    - 3 minutes
    - Always On (never turn off)
  </Accordion>
  
  <Accordion title="Wake Behavior">
    **Control when screen wakes up:**
    
    **Wake with All Events:**
    - Screen wakes for any alert
    - Includes non-critical notifications
    - Good for staying informed
    
    **Wake with Critical Events Only:**
    - Screen only wakes for important alerts
    - Disengagements, warnings, errors
    - Minimizes distractions
    
    **Manual Wake:**
    - Tap screen to wake
    - Press steering wheel button
  </Accordion>
</AccordionGroup>

## Configuration Tips

<AccordionGroup>
  <Accordion title="Optimize for Highway Driving">
    **Recommended settings:**
    
    - Enable Road Name display
    - Show Speed Limit with offset
    - Enable Lead Departure Alert
    - Display both distance and speed on chevron
    - Keep screen always on
  </Accordion>
  
  <Accordion title="Optimize for City Driving">
    **Recommended settings:**
    
    - Enable Turn Signals on screen
    - Show Standstill Timer
    - Enable Green Light Indicator
    - Show true speed (wheel sensors)
    - Use shorter interactive timeout
  </Accordion>
  
  <Accordion title="Minimize Distractions">
    **Clean UI setup:**
    
    - Disable non-essential visual elements
    - Use Screen Off While Driving
    - Enable Quiet Mode
    - Wake with critical events only
    - Hide speedometer if desired
  </Accordion>
  
  <Accordion title="Maximum Information">
    **Power user setup:**
    
    - Enable Developer UI
    - Show all metrics on chevron
    - Display road name
    - Show speed limit with offset
    - Enable all visual indicators
    - Use always-on screen
  </Accordion>
