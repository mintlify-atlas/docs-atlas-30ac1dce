## Overview

Sunnypilot's modular model backend (modeld_v2) allows you to select from over 86 different driving models, from Night Strike (October 2023) through The Cool People's Models (October 2025). Switch models instantly without reinstalling branches, download on-demand, and maintain a cache of your favorites.

<Info>
The model management system represents a complete rewrite of openpilot's model infrastructure, supporting modular runners (SNPE, thneed, tinygrad), dynamic inputs, and enhanced model outputs.
</Info>

## Switching Models

### From Settings UI

<Steps>
  <Step title="Open Models Panel">
    Navigate to Settings → Software → Current Driving Model → Change Model
    
    Or: Settings → Models
  </Step>
  
  <Step title="Search or Browse">
    Use search to find a specific model or browse the complete list
  </Step>
  
  <Step title="Select Model">
    Tap on the model you want to use
  </Step>
  
  <Step title="Download (if needed)">
    If not cached, model will download automatically. Progress is shown on screen.
  </Step>
  
  <Step title="Activate Model">
    Model switches immediately after download. Reboot may be required for some models.
  </Step>
</Steps>

<Warning>
**Internet Connection Required:**

Model downloads require an active internet connection. Each model is 40-100MB. Use WiFi on metered connections to avoid data charges.
</Warning>

### Technical Implementation

**Model execution pipeline:**

```python
class ModelState(ModelStateBase):
    def __init__(self):
        # Initialize model runner (SNPE/thneed/tinygrad)
        self.model_runner = get_model_runner()
        self.constants = self.model_runner.constants
        
        # Load active model bundle
        model_bundle = get_active_bundle()
        self.generation = model_bundle.generation
        
        # Setup buffers and inputs
        self.warp = Warp(buffer_length)
        self.temporal_buffers = {}
        self.numpy_inputs = {}
    
    def run(self, bufs, transforms, inputs, prepare_only):
        # Process vision inputs
        imgs_tensors = self.warp.process(bufs, transforms)
        
        # Prepare model inputs
        self.model_runner.prepare_inputs(self.numpy_inputs)
        
        # Run inference
        outputs = self.model_runner.run_model()
        
        # Update temporal buffers
        # Return model outputs
        return outputs
```

### Model Inputs

**Vision inputs:**
- Road camera (wide or standard)
- Additional camera streams (big_input)
- Warped and normalized images

**Policy inputs:**
- Desire (lane change intent)
- Traffic convention (LHD/RHD)
- Lateral control params (v_ego, delay)
- Features buffer (temporal history)
- Previous desired curvature

### Model Outputs

**Standard outputs:**
- Plan (trajectory)
- Lane lines
- Road edges  
- Lead detection
- Pose estimation
- Hidden state

**Enhanced outputs (model-dependent):**
- **Planplus:** Alternative trajectory for blending
- **Turn desires:** Recommended lane change direction
- **Desired curvature:** Direct curvature output

## Advanced Features

### Planplus Support

<Accordion title="What is Planplus?">
  **Alternative trajectory blending:**
  
  Some models output both a primary plan and a "planplus" alternative:
  
  ```python
  # Blend plans for smoother control
  curvature_plan = plan + (PLANPLUS_CONTROL - 1.0) * planplus
  ```
  
  **Configuration:**
  
  Settings → Developer → PlanplusControl
  
  **Values:**
  - `0.0`: Use only standard plan
  - `1.0`: Use blended approach (default)
  - Custom: Adjust blend ratio
</Accordion>

### Live Delay Adjustments

**Latency compensation:**

Models can adjust for system latency in real-time:

- **Lateral delay:** Steering actuation lag
- **Longitudinal delay:** Acceleration/brake lag
- **Software delay:** Processing time
- **Live updates:** Adjust based on measured latency

**Configuration:**

Settings → Developer → Live Delay

**Trade-offs to consider:**

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model won't download">
    **Check these:**
    
    - Internet connectivity (WiFi recommended)
    - Available storage space
    - Network not metered (or bypass warning)
    - Server availability
    
    **Solutions:**
    
    1. Verify internet connection
    2. Free up space (Settings → Storage)
    3. Retry download
    4. Check for software updates
    5. Try different model
  </Accordion>
  
  <Accordion title="Model crashes or errors">
    **Possible causes:**
    
    - Incompatible model for your hardware
    - Corrupted download
    - Insufficient resources
    - Model metadata mismatch
    
    **Debugging:**
    
    1. Check error logs (Settings → Developer)
    2. Delete and re-download model
    3. Try cache refresh
    4. Reboot device
    5. Switch to known-good model
    
    **Logs to check:**
    ```bash
    /data/community/crashes/modeld.log
    ```
  </Accordion>
  
  <Accordion title="Model switch doesn't take effect">
    **Verify:**
    
    - Download completed to 100%
    - No error messages
    - Correct model shown as active
    - Rebooted if required
    
    **Force switch:**
    
    1. Note desired model name
    2. Switch to different model
    3. Reboot device
    4. Switch back to desired model
    5. Confirm activation
  </Accordion>
  
  <Accordion title="Cache refresh not working">
    **Steps:**
    
    1. Navigate to Models panel
    2. Tap "Refresh Cache"
    3. Wait for rebuild (may take 30+ seconds)
    4. Check for completion message
    5. Browse models to verify
    
    **Manual cache clear:**
    ```bash
    # SSH to device
    rm -rf /data/models/cache/
    reboot
    ```
  </Accordion>
</AccordionGroup>
