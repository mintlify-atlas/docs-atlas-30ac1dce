## What is a Brand Port?

A brand port is a port of openpilot to a substantially new car brand or platform within a brand. Brand ports are more complex than model ports because you need to reverse engineer the car's CAN bus protocols and implement all the necessary interfaces from scratch.

<Info>
Example brand port: [Honda Clarity Port PR #23331](https://github.com/commaai/openpilot/pull/23331)
</Info>

## Step 1: Reverse Engineering

### Systematic Testing Protocol

Follow the stimulus-response test protocol to identify CAN bus signals. Each test should be done in a separate route (ignition cycle).

<Warning>
Constant power to the comma device is highly recommended using comma power. If constant power isn't available, keep the ignition on for at least one minute after your test to prevent data loss.
</Warning>

### Stationary Ignition Tests - Part 1

<Accordion title="Door and Seatbelt Signals">
1. Ignition on, but don't start engine, remain in Park
2. Open and close each door in order: driver, passenger, rear left, rear right
3. Re-enter the vehicle, close the driver's door, and fasten the driver's seatbelt
4. Slowly press and release the accelerator pedal 3 times
5. Slowly press and release the brake pedal 3 times
6. Hold the brake and move the gearshift: reverse â†’ neutral â†’ drive â†’ sport/eco (if applicable)
7. Return to Park, ignition off

**What to look for:**
- Brake-pressed information (may appear in multiple messages)
- All four door states (some cars only expose driver's door on ADAS bus)
- Seatbelt status
- Gear shifter positions
</Accordion>

### Stationary Ignition Tests - Part 2

<Accordion title="ACC Buttons and Blinkers">
1. Ignition on, but don't start engine, remain in Park
2. Press each ACC button in order: main on/off, set, resume, cancel, accel, decel, gap adjust
3. Set the left turn signal for about five seconds
4. Operate the left turn signal in touch-to-pass mode
5. Set the right turn signal for about five seconds
6. Operate the right turn signal in touch-to-pass mode
7. Set the hazard/emergency indicator for about five seconds
8. Ignition off

**What to look for:**
- ACC button press signals
- Turn signal and hazard states
</Accordion>

### Steering Tests

<Accordion title="Steering Angle and Torque">
Power steering should be available. On ICE cars, engine RPM may be present.

1. Ignition on, start engine if applicable, remain in Park
2. Rotate the steering wheel with pauses between each step:
   - Start at exact center
   - Turn to 45Â° right and hold
   - Turn to 90Â° right and hold
   - Turn to 180Â° right and hold
   - Turn to full lock right and hold with firm pressure
   - Release the wheel and allow it to bounce back slightly
   - Turn to 180Â° left and hold
   - Return to center and release
3. Ignition off

**What to look for:**
- Steering wheel angle signals
- Driver input torque signals
- Signal scale and sign/direction confirmation
</Accordion>

### Low Speed Driving Tests

<Accordion title="Parking Lot Maneuvers">
Drive to an empty parking lot where you can safely drive in curves.

1. Ignition on, start engine, prepare to drive
2. Slowly (10-20mph max) drive a figure-8, or at least one sharp left and one sharp right
3. Come to a complete stop
4. When safe, drive in reverse for a short distance (10-15 feet)
5. Park the car safely, ignition off

**What to look for:**
- Wheel speed signals
- Vehicle speed
- Reverse gear detection
- Standstill detection
</Accordion>

### High Speed Driving Tests

<Accordion title="Highway Testing">
Select a safe location where you can test cruise control and lane guidance at highway speeds.

1. Ignition on, start engine, prepare to drive
2. When safe, engage adaptive cruise control below 50 mph
3. Use ACC buttons to accelerate: 50mph â†’ 55mph â†’ 60mph
4. Disengage adaptive cruise
5. When safe, allow factory lane guidance to prevent lane departures 2-3 times on both left and right

**What to look for:**
- ACC cruise setpoint signals
- ACC HUD signals and scaling
- Lane keep assist steering commands
- Confirmation of wheel speed scaling
</Accordion>

## Step 3: Implement Core Interfaces

### Required CarState Signals

<AccordionGroup>
  <Accordion title="Basic Lateral Control Signals (Required)">
    - `brakePressed`
    - `cruiseState`
    - `doorOpen`
    - `espDisabled`
    - `gasPressed`
    - `gearShifter`
    - `leftBlinker` / `rightBlinker`
    - `seatbeltUnlatched`
    - `standstill`
    - `steeringAngleDeg`
    - `steeringPressed`
    - `steeringTorque`
    - `steerFaultPermanent`
    - `steerFaultTemporary`
    - `vCruise`
    - `wheelSpeeds.[fl|fr|rl|rr]`: Speed of each wheel in m/s
  </Accordion>

  <Accordion title="Longitudinal Control Signals (Required for openpilot longitudinal)">
    - `accFaulted`
    - `espActive`
    - `parkingBrake`
  </Accordion>

  <Accordion title="Automatically Populated Signals">
    These are populated automatically by `parse_wheel_speeds`:
    - `aEgo`: Acceleration (Kalman filtered derivative of vEgo)
    - `vEgo`: Vehicle speed (Kalman filtered from vEgoRaw)
    - `vEgoRaw`: Unfiltered speed from average of four wheels
    - `buttonEvents`: Button events
  </Accordion>

  <Accordion title="Optional Signals">
    - `brake`
    - `charging`
    - `fuelGauge`
    - `leftBlindspot` / `rightBlindspot`
    - `steeringRateDeg`
    - `stockAeb`
    - `stockFcw`
    - `yawRate`
  </Accordion>
</AccordionGroup>

### interface.py Template

<CodeGroup>
```python interface.py
from opendbc.car import CarInterfaceBase
from opendbc.car.interfaces import RadarInterfaceBase

class CarInterface(CarInterfaceBase):
  @staticmethod
  def get_params(candidate, fingerprint=None, car_fw=None):
    # Define car parameters, limits, and capabilities
    ret = CarInterfaceBase.get_std_params(candidate)
    
    # Set steering limits
    ret.steerLimitAlert = True
    ret.steerActuatorDelay = 0.1  # seconds
    
    # Set safety model
    ret.safetyModel = car.CarParams.SafetyModel.[brand]
    
    return ret

  def update(self, c, can_strings):
    # Update vehicle state from CAN messages
    return self.CS.update(self.cp, self.cp_cam)

  def apply(self, c):
    # Apply controls to vehicle
    return self.CC.update(c.enabled, self.CS, c.actuators)
```

```python carstate.py
from opendbc.can.parser import CANParser

class CarState:
  def update(self, cp, cp_cam):
    ret = car.CarState.new_message()
    
    # Parse CAN signals
    ret.vEgo = cp.vl["WHEEL_SPEEDS"]["FL"] * CV.KPH_TO_MS
    ret.steeringAngleDeg = cp.vl["STEERING_ANGLE"]["ANGLE"]
    ret.brakePressed = cp.vl["BRAKE_STATUS"]["PRESSED"] == 1
    
    # ... parse other signals
    
    return ret
```

```python carcontroller.py
from opendbc.car.[brand].[brand]can import create_steer_command

class CarController:
  def update(self, enabled, CS, actuators):
    can_sends = []
    
    # Apply steering
    if enabled:
      steer_cmd = create_steer_command(
        actuators.steer,
        CS.steeringAngleDeg
      )
      can_sends.append(steer_cmd)
    
    return can_sends
```
</CodeGroup>

## Step 4: Implement Safety Code

Create safety logic in `opendbc_repo/opendbc/safety/modes/[brand].h`:

<Accordion title="Safety Code Requirements">

**Critical Safety Checks:**

1. **Steering Rate Limits**: Prevent sudden steering changes
2. **Torque Limits**: Constrain maximum steering torque
3. **Driver Override**: Allow immediate driver takeover
4. **Speed-Dependent Limits**: Adjust limits based on vehicle speed
5. **Fault Detection**: Detect and handle system faults

**Safety Test Suite:**

Create comprehensive tests in `opendbc_repo/opendbc/safety/tests/test_[brand].py`:

- Test all safety limits
- Test driver override scenarios
- Test fault conditions
- Ensure 100% code coverage

<Warning>
**All safety tests must pass.** Forks with failing safety tests will be banned from comma.ai servers.
</Warning>
</Accordion>

## Common Pitfalls

<Warning>
- **Incomplete signal mapping**: Missing critical CarState signals
- **Incorrect scaling**: Wrong units for speed, angle, or torque
- **Safety limits too aggressive**: Actuation changes too fast for safe driver reaction
- **Insufficient testing**: Not validating all scenarios
- **Missing edge cases**: Not handling fault conditions
</Warning>
